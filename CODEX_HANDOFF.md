Recommended Tilda/embed wrapper generated by the configurator:

```html
<div style="width:100%;height:78vh;min-height:720px;position:relative;overflow:hidden;">
  <iframe src="https://webgl-viewer-jet.vercel.app/?player=1&scene=https%3A%2F%2Fwebgl-viewer-jet.vercel.app%2Fscenes%2Fthelordoftherings%2Fscene.json&transparent=1" style="width:100%;height:100%;border:0;display:block;background:transparent;" allow="autoplay; fullscreen; accelerometer; gyroscope; magnetometer" allowtransparency="true" scrolling="no"></iframe>
</div>
```

Important:

- The iframe height must be controlled by the wrapper, not by a square/default iframe box.
- For desktop, `min-height:720px` was needed to stop top cropping on the ring scene.
- For mobile, the same AUTO scene looked good because the container aspect was tall.
- If the desktop page crops again, first check Tilda wrapper height and scene camera, not the GLB.

## Frame Format / AUTO Behavior

User-confirmed rule:

- `Frame Format = AUTO` is the normal mode for responsive iframe publishing.
- AUTO means the published player uses the actual iframe/container aspect.
- Separate fixed formats are only needed when the user wants different camera compositions for specific social/video ratios.

Important implementation details:

- `src/components/Viewport.tsx` keeps the Canvas stable when toggling `Show Frame Guides`.
- `CameraBridge` syncs camera aspect from the actual R3F size.
- `Show Frame Guides + AUTO` must not shrink/remount the scene into a square.
- `src/components/TransparentPublishedViewport.tsx` also syncs camera aspect.
- `src/app/PublishedPlayerApp.tsx` has a safe desktop fallback for AUTO published cameras.

If a newly published scene crops on desktop:

1. Open the scene in the editor.
2. Use `Frame Format = AUTO`.
3. Turn on `Show Frame Guides` only as a guide, not as a fixed output frame.
4. Set camera composition so it fits the wide desktop frame.
5. Publish again to the same scene slug.
6. Wait for Vercel deployment and hard refresh the portfolio page.

## Publish Workflow

The `WEB` button publishes a scene package into:

- `public/scenes/<scene-slug>`

Server/plugin code:

- `vite.config.ts`

Client publish code:

- `src/features/publish/exportWebPackage.ts`

The publish dialog now:

- lists existing scenes;
- defaults to overwriting the latest scene;
- allows selecting an existing scene to replace;
- allows typing a new scene name;
- sanitizes scene slugs.

Publish behavior:

1. Runs a production build check.
2. Writes or replaces `public/scenes/<scene-slug>`.
3. Stages only that scene folder.
4. Commits with `Publish scene <slug>` or `Update scene <slug>`.
5. Pushes the current branch to GitHub.
6. Vercel deploys from GitHub.

Important:

- Scene publishing and player/app deployment are different things.
- Runtime/player code fixes need a normal app commit/push.
- Scene content/camera changes need republishing the scene.

`SCNS` opens the published scene manager.

- It is no longer rendered inside the left sidebar.
- It shows published scenes newest-first.
- It has a close `X`.
- Each scene row has an `X` delete button.

Delete behavior:

1. Deletes `public/scenes/<scene-slug>` locally.
2. Runs `git add -- public/scenes/<scene-slug>`.
3. Verifies no unrelated staged files are present.
4. Commits `Delete scene <scene-slug>`.
5. Pushes to GitHub.
6. Vercel removes the scene on the next deployment.
